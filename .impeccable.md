## Design Context

### Users
Software developers, educators, technical content creators, and students who need to share code snippets in presentations, social media, documentation, or forums. They value speed, lack of friction (no login/backend), and high-quality visual output.

### Brand Personality
Minimalist, efficient, frictionless, and professional. The tool should feel like a fast, native utility rather than a heavy web app.

### Aesthetic Direction
Utilitarian and highly minimal. Exclusively uses a grayscale palette for the UI (with specific light and dark mode tokens) to ensure the code snippet itself remains the focal point. Typography relies on clean sans-serifs (Inter/system-ui) for the UI and crisp monospaced fonts (JetBrains Mono/Fira Code) for the code. No unnecessary borders, shadows, or decorations.

### Design Principles
1. **Content First:** The code snippet is the hero; UI elements must be completely secondary and visually recede (grayscale palette). **Note: Syntax highlighting themes for the code snippet (e.g., Monokai, Solarized) are exempt from the grayscale rule as they constitute the "content" the user is creating.**
2. **Zero Friction:** Instant visual feedback (real-time preview) with no unnecessary steps or configuration to get a downloadable image.
3. **Ruthless Minimalism:** Remove any element that doesn't directly serve the core function of configuring or capturing the snippet. Every visible component must have a clear purpose. **Do not remove functional syntax highlighting themes; minimalism applies to the shell/interface, not the core content capabilities.**
